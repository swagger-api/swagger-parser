## Analysis Files

Each issue has a corresponding analysis file following the naming convention:
```
parser-batch4-issue-{number}.md
```

For example:
- `parser-batch4-issue-1147.md`
- `parser-batch4-issue-1149.md`
- etc.

## Analysis Template

All analysis files are generated using the prompt template located at:
```
.github/analyze-issue.md
```

## Structure

Each analysis file includes:
1. Issue Details (number, title, state, dates, URL, labels)
2. Issue Body
3. Analysis Framework with 7 sections:
   - Issue Classification
   - Technical Summary
   - Root Cause Analysis
   - Impact Assessment
   - Proposed Solution
   - Implementation Considerations
   - Additional Notes

## Usage

These analysis files serve as templates for conducting thorough issue analysis. To complete the analysis:

1. Open an analysis file (e.g., `parser-batch4-issue-1147.md`)
2. Review the issue details and body
3. Fill in each section of the Analysis Framework
4. Save the completed analysis
